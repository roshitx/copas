# Task 9: Live Twitter Smoke Test Suite

## Summary
Created live Twitter smoke test suite for copas.io that uses real network calls against the 5 canonical Twitter URLs.

## Files Created

### 1. frontend/tests/e2e/twitter-live-smoke.spec.ts
Playwright E2E test suite with:
- @live tag for selective execution
- Real network calls (no stubbing)
- 60s timeout per test (live network resilience)
- 2 retries for flaky network handling
- Screenshots captured to .sisyphus/evidence/ on each test
- Tests all 5 canonical scenarios:
  - single image: https://x.com/rwhendry/status/2027767749695705372?s=20
  - single video: https://x.com/sosmedkeras/status/2027955413753417803?s=20
  - multi-video: https://x.com/mikuroQ/status/2027735620534358393?s=20
  - multi-image: https://x.com/IndonesiaGaruda/status/2027914018976108959?s=20
  - hybrid: https://x.com/Villgecrazylady/status/2027532953966825726?s=20

### 2. backend/tests/live/test_twitter_live.py
pytest live smoke test suite with:
- @pytest.mark.live marker
- Direct extract_media_info() calls
- 30s timeout per test
- Short assertions: returns MediaResult, has title, has formats > 0
- No brittle label/count checks (Twitter can change)

## Execution Commands

### Frontend (Playwright)
```bash
# Run live tests only
npx playwright test --grep @live

# Exclude live tests (deterministic only)
npx playwright test --grep-invert @live
```

### Backend (pytest)
```bash
# Run live tests only
pytest -m live

# Exclude live tests
pytest -m "not live"
```

## Design Decisions

1. **Separate from deterministic suite**: Live tests do NOT run on every PR
2. **Minimal assertions**: Only check structure, not exact counts/labels
3. **Timeouts**: Longer timeouts for live network (30-60s)
4. **Retries**: 2 retries for network resilience
5. **Evidence capture**: Screenshots saved for debugging
