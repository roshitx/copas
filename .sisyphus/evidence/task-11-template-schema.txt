### Pattern Validation
- ✓ Scenario ID format: `{platform}-{scenario-type}`

- Instagram uses `/p/` for posts and `/reel/` for videos

## Validation Notes

### Platform Template Usage
To use this template for a new platform:
1. Copy `tests/shared/platform-template.json` to `{platform}-scenarios.json`
2. Replace all "instagram" with new platform name
3. Update URLs to valid platform URLs
4. Update author handles to real accounts
5. Adjust scenario counts and types as needed
6. Validate against `tests/shared/scenario-schema.json`

### Documentation Usage
To add a new platform using the documentation:
1. Follow Step 1-8 in order
2. Use required field tables to create fixtures
3. Copy assertion patterns for each scenario type
4. Follow naming conventions for all files
5. Use validation checklist before submitting

### CI Integration
Documentation includes CI workflow examples for:
- Backend tests: `pytest -m "integration or contract" -k {platform}`
- Frontend tests: `npx playwright test tests/e2e/{platform}-deterministic.spec.ts`
- Live smoke: `@live` marker integration
