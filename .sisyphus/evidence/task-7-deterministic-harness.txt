# Task 7: Deterministic Playwright E2E Harness

## Summary
Built the deterministic Playwright E2E harness for the copas.io frontend. The harness intercepts all backend API calls via page.route() ensuring NO real network calls happen to X.com / fxtwitter / yt-dlp.

## Files Created/Modified

### 1. frontend/tests/e2e/twitter-deterministic.spec.ts
- Contains harness setup + all 5 Twitter scenario assertions
- Uses fixtures helper instead of direct JSON imports to avoid module resolution issues
- Tests for:
  - Single Image scenario (1 format button, IMAGE badge, flat list)
  - Single Video scenario (1 format button, VIDEO badge, duration)
  - Multi-Video scenario (2 videos, accordion groups, VIDEO badges)
  - Multi-Image scenario (2 images, Download Semua button, IMAGE badges)
  - Hybrid Video+Image scenario (both sections, both badges)

### 2. frontend/tests/e2e/helpers/fixtures.ts
- Loads JSON fixtures using fs.readFileSync
- Exports individual fixtures for all 5 scenarios:
  - singleImageFixture
  - singleVideoFixture
  - multiVideoFixture
  - multiImageFixture
  - hybridFixture

### 3. frontend/tests/e2e/helpers/assertions.ts
- Reusable assertion helpers for UI testing
- Includes functions for:
  - waitForResultCard
  - assertFormatButtonCount
  - assertMediaTypeBadges
  - assertDownloadAllButtonVisible
  - assertVideoAccordionGroups
  - assertHybridSections
  - createNetworkInterceptor

### 4. frontend/tests/e2e/helpers/mock-api.ts (Enhanced)
- mockExtractApi() - Stubs POST /api/extract with fixture data
- mockDownloadApi() - Stubs GET /api/download with fake blobs
- createDownloadInterceptor() - Captures download requests
- mockAllExternalCalls() - Blocks all external domains (x.com, twitter.com, fxtwitter, yt-dlp)
- transformFixture() - Converts camelCase fixtures to snake_case API format

### 5. frontend/components/result-card.tsx (Modified)
Added data-testid attributes for stable selectors:
- data-testid="result-card" - Main result card container
- data-testid="video-section" - Videos accordion section
- data-testid="image-section" - Images accordion section
- data-testid="audio-section" - Audio accordion section
- data-testid="video-group" - Per-video accordion groups
- data-testid="download-all-button" - Download Semua button
- data-testid="format-list" - Flat format list (non-accordion)

### 6. frontend/components/format-button.tsx (Modified)
Added data-testid="format-button" to each format button for easy selection.

## Key Features

### API Interception
All backend API calls are intercepted:
- POST /api/extract - Returns fixture JSON based on URL
- GET /api/download?token=... - Returns fake binary blobs

### Zero External Network Calls
The mockAllExternalCalls() function blocks:
- x.com / twitter.com
- fxtwitter / vxtwitter
- yt-dlp
- youtube.com / youtu.be
- Any other external domains (only localhost:3000 and localhost:8000 allowed)

### Test Scenarios Covered
1. **Single Image**: 1 format button, IMAGE badge visible, no accordion
2. **Single Video**: 1 format button, VIDEO badge, duration displayed, video section visible
3. **Multi-Video (2)**: 2 format buttons, VIDEO badges, 2 video groups in accordion
4. **Multi-Image (2)**: 2 format buttons, IMAGE badges, Download Semua button visible
5. **Hybrid (Video+Image)**: Both video and image sections, both VIDEO and IMAGE badges

## Running the Tests
```bash
cd frontend
npx playwright test tests/e2e/twitter-deterministic.spec.ts --reporter=list
```

## Notes
- The test file uses the fixtures helper to load JSON files to avoid TypeScript/ESM import issues
- All fixtures are located in frontend/tests/fixtures/twitter/
- The harness ensures deterministic, repeatable tests with no external dependencies
- Tests verify both UI structure (buttons, badges, sections) and download functionality
